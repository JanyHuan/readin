# 读书笔记网站化 · 方案介绍

> 目标：把飞书多维表格里的读书笔记，做成一个**可时时更新、简洁美观、有高级感**的网页。

## 一、技术方案：如何让笔记「时时更新」

数据继续留在飞书多维表格里（你照常在飞书写），网页通过飞书开放接口自动同步，无需手动搬运。

同步架构（见示意图）：
1. **数据源**：飞书多维表格（书名、作者、笔记、标签…）
2. **服务端代理**（Serverless 函数）：用 app_id + app_secret 换取 tenant_access_token，调用 Bitable OpenAPI 拉取记录，做字段整理与缓存
3. **JSON 接口**：返回给前端的标准化数据（带短缓存，如 60s）
4. **前端静态页**：纯 HTML/CSS/JS，打开即拉取最新 JSON 并渲染
5. **用户浏览器**：实时看到最新笔记

两种同步策略：
- **实时拉取（推荐）**：每次打开页面由代理拉最新，天然「时时更新」，几乎零延迟感
- **Webhook 主动刷新（进阶）**：表格变更时飞书推事件，触发重新生成/刷新，更省调用额度

## 二、页面布局设计（见布局预览图）

整体结构：
- **顶栏**：站点名（如「读书手账」）+ 全局搜索 + 视图切换（网格/列表/时间线）+ 明暗主题切换
- **筛选区**：状态（想读/在读/读完）、标签、评分、年份，以 chip 形式横向排列或左侧栏
- **主体**：卡片网格，每张卡片 = 一本书

### 卡片展开 / 收起交互（重点）
提供三种由浅入深的交互，可组合：
1. **行内展开（Accordion）**：卡片默认只显示封面、书名、作者、评分、标签与一句摘录；点击「展开」后下方平滑展开完整读书笔记、金句摘录、阅读日期，再点收起。适合快速浏览。
2. **详情抽屉 / 浮层（Slide-over / Modal）**：点击卡片主体，从右侧滑出或居中弹出详情面板，展示大封面 + 完整笔记 + 多条摘录 + 相关标签。背景轻遮罩，点空白或按钮关闭。适合沉浸式阅读单本。
3. **网格就地放大（FLIP 动画）**：在网格视图下点某张卡片，该卡片就地放大占据更大面积并展开内容，其余卡片自动重排让位；再点还原。视觉最「高级」，实现成本最高。

> 建议：V1 用「行内展开 + 详情浮层」组合，体验完整又不复杂；FLIP 放大作为 V2 增强。

### 多视图
- **网格视图**：错落卡片墙，重视觉
- **列表视图**：紧凑一行一书，重信息密度（适合检索）
- **时间线视图**：按阅读/记录时间纵向铺开，像一条「阅读轨迹」

## 三、UI 设计风格（三种方向，待你选）

1. **文艺编辑风（浅色）** — 米白纸张质感 + 衬线标题（如思源宋体 / Noto Serif），温润、像精装读书笔记，最「耐看」。
2. **暗色高级风** — 深空背景 + 克制留白 + 极少量强调色，像 premium 阅读应用，科技感、聚焦内容。
3. **画廊网格风** — Are.na / Pinterest 式错落卡片墙，强调封面与视觉节奏，适合笔记本身很有设计感。

通用原则：大量留白、≤2 种主色、衬线/无衬线搭配、微动效（展开约 200ms 缓动）、响应式（手机单列、桌面多列）、支持明暗主题。

## 四、假设的数据字段
书名 · 作者 · 封面图 · 状态（想读/在读/读完）· 评分（1-5）· 标签 · 读书笔记（长文）· 金句摘录 · 开始/读完日期。
（若你的表结构不同，告诉我实际字段即可对齐。）

## 五、技术栈建议
- 前端：静态 HTML/CSS/JS（可上 Vanilla / Vite / Next.js 静态导出）
- 同步：Serverless 函数（EdgeOne Pages Functions / Vercel Functions / Cloudflare Workers）调用 Feishu OpenAPI
- 部署：EdgeOne Pages / Vercel（免费额度够个人用，全球加速）
